大模型使用的三个层次

提示词工程 · 上下文工程 · Agent 工程

切入点 「1 + 1 等于几？」
目标受众 有一定 AI 使用经验的学习者
核心目标 理解三层递进关系
课堂导入
1 + 1 等于几？
先别急着回答——我们用这个问题来理解三种不同的思维层次。
三个层次 · 逐层深入
1
提示词工程
Prompt Engineering
核心问题：我怎么问，才能得到更好的答案？
关注的是单次输入如何写得更精准、更有效。技巧在于措辞、格式、角色设定。
弱提示
「1 + 1 等于几？」
模型答：2。对，但没什么用。
强提示
「你是一位小学数学老师，用孩子能听懂的方式解释 1 + 1 = 2。」
模型现在给出了类比、解释、例子——因为角色 + 格式 + 目标都在提示里了。
💡 提示词工程的本质：你在一句话里压缩了足够多的信息，让模型猜对你的意图。
2
上下文工程
Context Engineering
核心问题：我怎么给模型提供更完整的世界？
不只是一句话，而是精心设计放进对话窗口里的所有信息——记忆、文档、历史、身份、状态。
场景
同样问「1 + 1 等于几？」——但上下文不同，答案完全不同：

· 二进制计算机科学上下文 → 1 + 1 = 10
· 布尔逻辑上下文 → 1 + 1 = 1（OR）或 0（XOR）
· 哲学上下文 → 整体大于部分之和
· 孩子的作业上下文 → 2，并配合图示说明
工程侧
把用户历史、RAG 检索结果、系统人设、当前任务状态全部注入上下文窗口——模型看到的世界由你来设计。
💡 上下文工程的本质：答案永远是对的，因为你给了模型一个完整的世界。
3
Agent 工程
Agent Engineering
核心问题：模型能不能自己去弄清楚 1 + 1 是什么意思，然后给我答案？
不再是你喂给模型信息，而是让模型主动感知、规划、调用工具、执行、反思。
执行链
Agent 接收模糊任务：「帮我算 1 + 1」

1. 感知：读取用户背景，判断这是数学题还是哲学题
2. 规划：分解步骤，决定是否调用计算器工具
3. 执行：调用工具，得到结果
4. 反思：答案对吗？用户满意吗？要不要追问？
5. 记忆：把这次偏好存下来，下次更准
关键变化
从「问答」变成了「协作」——模型从被动回答者变成主动执行者，上下文由模型自己去动态构建。
💡 Agent 工程的本质：你不再设计答案，而是设计一个会自己找答案的系统。
横向对比
维度 提示词工程 上下文工程 Agent 工程
操作者 人精心写提示 系统注入上下文 模型自主行动
信息来源 你的一句话 预设的知识库/历史 模型动态获取
交互次数 一次 一次（丰富输入） 多轮自主循环
类比 聪明地提问 给足参考资料再问 交给助手自己去办
「1+1」的答案 取决于怎么问 取决于你在哪个世界 模型自己判断你要什么
课堂讨论点
三层递进，不是替代：好的 Agent 系统里，仍然需要优质的提示词和精心的上下文设计——它们是地基，不是被淘汰的旧技术。
「1+1」的比喻价值：同一个问题在不同层次下会有不同答案，说明模型不是查字典——它是在根据你提供的世界来推理。
学习路径建议：从提示词工程开始打基础 → 学会上下文设计来构建产品 → 掌握 Agent 工程来解决复杂任务。
